The Cataloging Calculator

by Kyle Banerjee

Start using the Calculator

System requirements

IE 5.5, Netscape 6.1, Opera 5.1 or better -- the Cataloging Calculator does not work with old versions of Netscape (such as 4.7) and is not tested with old versions of IE. Users with older and overloaded machines may experience problems with their browsers freezing up or not responding to searches.

Note that the Cataloging Calculator takes bit of time to load because it retrieves a number of large data files when the page is first opened. You cannot perform your first search until a message appears in the top frame. Once the calculator is loaded, it should work well.

Using the Cataloging Calculator

The display consists of three frames. The top one displays search results, the middle one contains the search options, and the bottom allows fixed field searches. Call number/subject heading correlations and MARC variable and fixed field information retrieved from OCLC are displayed in a separate window. There is no need to close the window because the Cataloging Calculator always uses the same one.

To search fixed field elements

  • Select the type of fixed field you want to search
  • Click on the appropriate element
  • The results will be retrieved from OCLC and displayed in a separate window.

To search LC or geographic Cutter numbers, geographic area codes, country codes, language codes, AACR2 abbreviations, or MARC variable fields (bibliographic only)

  • Select click on the search type desired
  • Type an appropriate search such as main entry, geographic entity, code, MARC tag, or keyword.
  • Results for LC Cutter numbers start appearing as soon as you make the first keystroke.
  • All other searches will display results interactively starting with the third keystroke.

If a number of MARC fields are displayed in the results pane in response to a keyword query, you may click on any of them to see the results in OCLC. If there is only result in response to a query, the window will automatically open. Users of slower computers (or overtaxed ones) may experience browser freeze ups when performing interactive MARC searches.
